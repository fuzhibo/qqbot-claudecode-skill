# /qqbot-check

统一检查 QQ Bot 可用状态，整合分散的状态检查功能。

## 用法

```bash
/qqbot-check [--receive] [--send] [--fix]
```

## 选项

- `--receive` - 检查消息接收能力（WebSocket 连接、后台服务）
- `--send` - 检查消息发送能力（API 连通性、凭证有效性）
- `--fix` - 显示自动修复建议（默认开启）
- 无选项 - 执行完整检查

## 检查内容

### 环境检查
- Node.js 版本 (>= 18.0.0)
- 必要依赖包 (ws, dotenv)
- 构建产物 (dist 目录、MCP 入口)

### 凭证配置
- 配置文件存在性
- 机器人配置完整性
- AppID 和 Client Secret
- 默认目标设置（可选）

### 发送能力
- Token API 连通性
- Message API 连通性
- 凭证有效性

### 接收能力
- 后台服务运行状态
- 项目注册状态
- 默认项目设置

## 使用场景

### 场景1: 使用前的快速检查
```bash
/qqbot-check
```
在使用 QQ Bot 前确认所有组件正常工作。

### 场景2: 排查接收问题
```bash
/qqbot-check --receive
```
当收不到 QQ 消息时，专门检查接收链路。

### 场景3: 排查发送问题
```bash
/qqbot-check --send
```
当发送消息失败时，专门检查发送链路。

### 场景4: 自动化脚本中使用
```bash
node scripts/qqbot-check.js
if [ $? -eq 0 ]; then
  echo "检查通过"
else
  echo "检查失败"
fi
```

## 异常处理

| 状态 | 图标 | 含义 | 处理方式 |
|------|------|------|----------|
| 通过 | ✅ | 正常 | 无需处理 |
| 警告 | ⚠️ | 可选功能缺失 | 可忽略或稍后处理 |
| 错误 | ❌ | 核心功能异常 | 必须修复才能使用 |

## 与现有指令的关系

| 指令 | 用途 | 对比 |
|------|------|------|
| `/qqbot-check` | 统一状态检查 | ✅ 推荐，一次检查所有 |
| `/qqbot-service status` | 仅服务状态 | 子集，不包含凭证/网络 |
| `/qqbot-doctor` | 环境诊断+修复 | 更底层，用于故障排查 |
| `/qqbot-list` | 查看机器人 | 仅显示凭证信息 |

## 执行

```bash
node ${CLAUDE_PLUGIN_ROOT}/scripts/qqbot-check.js ${args}
```

## 提示

- 建议在新项目首次使用前运行 `/qqbot-check`
- 收不到消息时，先运行 `/qqbot-check --receive`
- 发送失败时，先运行 `/qqbot-check --send`
- 检查结果中的警告项可忽略，但错误项必须修复
